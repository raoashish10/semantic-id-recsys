RecSys Demo Panel

RQ-VAE + SASRec · Redis · FastAPI · localhost:8000
What this is
A production-style recommendation system built on the Amazon Beauty dataset. Items are encoded into semantic ID codes (RQ-VAE), and a sequential model (SASRec) predicts what a user will want next based on their browsing history.
Three serving paths
Cache hit — recs precomputed offline, served from Redis instantly.
Cold start — new user with <3 items, uses category prefix lookup.
Warm user — SASRec runs live beam search over semantic ID space.
How to use this panel
Click a scenario button to pre-fill a real example, then hit Get Recommendations. The badge and latency show which path served the response. Each recommendation includes its semantic ID tokens — the discrete codes that represent it in the model.
